  • 题解

    关于这题有些人的方法为:
    $$
    sum=a+b
    $$
    该方法也能通过编译，但是不符合题目要求，这题本质还是落在的位运算上。

    首先我们先明确几个概念：

    编译器位数 char short int int long long long float double char *
    32位(4byte) 8(1) 16(2) 32(4) 32(4) 64(8) 32(4) 64(8) 16(2)
    64位(8byte) 8(1) 16(2) 32(4) 64(8) 64(8) 32(4) 64(8) 64(8)

    这里我们需要知道对于unsigned类型，其实就是将最高位变成符号位，不影响整个数据类型的长度。

    那么这道题我们用到的是int，现在一般机器都是64位，所以长度位32位。但是32位太长了，我们这里考虑8位如何进行加法的位运算。

    • a^b,异或运算是进行无进位的加法运算

      a：0000 0101

      b：0000 0010

      a^b：0000 0111

    • (a&b)<<1 获取进位

      a：0000 1111

      b：0000 0010

      (a&b)<<1：0000 0010<<1 0000 0100

    • 考虑是否溢出

      本题提示了：-1000 <= a, b <= 1000，所以不存在溢出的问题。
